# Production Handoff v1

## 目的
- `index.html` の構造を大きく変えずに、本番反映時の差し替え箇所を明確にする
- 画像、福利厚生文言、キャリアパス文言、CTA導線先の反映準備を整理する

## 1. 画像差し替えポイント

### ヒーロー背景画像
- ファイル: `index.html`
- 位置目安: ヒーローセクション内の最初の `img`
- 現在値:
  - `src="https://images.unsplash.com/photo-1517048676732-d65bc937f952?..."`
  - `alt="技術チームの打ち合わせ風景"`
- 差し替え内容:
  - 会社の雰囲気や技術現場に合う正式画像
- 確認事項:
  - 文字が読める明度か
  - スマホで人物や設備が切れすぎないか
  - 技術軽視や過度な華美さに見えないか

### 事業領域画像
- ファイル: `index.html`
- 対象:
  - Mobility
  - Life Science
  - Manufacture
- 現在値:
  - 3枚とも Unsplash 仮画像
- 差し替え内容:
  - 各事業領域に対応する正式画像
- 確認事項:
  - 領域と画像内容が一致しているか
  - 3枚のトーンに統一感があるか
  - 画像トリミング後も意味が伝わるか

## 4. CTA導線差し替えポイント

### メインCTA
- ファイル: `index.html`
- セクション: `#cta`
- 現在リンク:
  - `mailto:info@e-gravity.co.jp?subject=まずは話を聞きたい`
  - `mailto:info@e-gravity.co.jp?subject=カジュアル相談`
  - `mailto:info@e-gravity.co.jp?subject=応募について`
- 差し替え内容:
  - 採用フォーム URL
  - カジュアル面談予約 URL
  - 問い合わせフォーム URL または採用LP URL
- 反映ルール:
  - 最優先導線を1つ決める
  - 3導線とも役割が重複しないようにする
  - 外部サービス利用時は別タブ遷移の要否を確認する

### ヘッダーCTA
- ファイル: `index.html`
- 現在リンク:
  - `href="#cta"`
- 差し替え候補:
  - 本番では最優先の採用導線に直接遷移させるか、現状通り CTA セクション誘導にするか判断

## 5. 実データ確認が必要な項目
- 正式画像の権利確認、使用可否、トリミング前提の確認
- 福利厚生の正式名称、掲載順、公開可能範囲
- キャリアパスの実在社員ベース年次と役割の整合性
- 社員ストーリーの匿名/実名運用、公開範囲、写真有無
- CTAの正式遷移先 URL と運用窓口
- 採用窓口メールアドレスが `info@` のままでよいかどうか

## 6. 本番反映前チェック
- 画像差し替え後にヒーロー文字の可読性を再確認
- CTA差し替え後に PC/スマホでボタン優先度が崩れていないか確認
- 福利厚生文言が実制度と一致しているか確認
- キャリアパスと社員ストーリーが誇張表現になっていないか確認
- すべてのリンク遷移先が意図通りか確認
